# 入職首季行動方案 (SRE/DevOps Onboarding 30-60-90 Day Plan)
這是我針對「雲端平台運維與治理」職位預擬的入職計畫。旨在快速融入團隊，並將我在專案中積累的治理經驗轉化為組織價值。

---

## 🟢 第一階段：融入與觀察 (Day 1 - 30)
**目標：熟悉現有架構、工具鏈與溝通流程。**
1.  **環境熟悉**：盤點現有的 AWS/GCP 資源分布，理解目前的 VPC 規劃、IAM 授權現況。
2.  **流程對齊**：熟悉團隊目前的 Jenkins/GitLab CI/CD 配置，觀察目前的上線檢核流程 (Checklist)。
3.  **監控接入**：掌握 Grafana 與告警系統（如 Slack/Nagios）的配置，理解目前定義的 SLA 與各應用負責窗口。
4.  **建立信任**：處理首批低風險的小型 Ticket，快速熟悉團隊的 Git 分支策略與變更流程。

## 🟡 第二階段：穩定與治理 (Day 31 - 60)
**目標：優化日常維運效率，導入「平台化」思維。**
1.  **文檔資產化**：根據團隊現狀更新 Runbook。若團隊缺乏 SOP，則導入我在專案中實踐的 **「事後分析回饋機制」**。
2.  **安全性加固**：針對現有叢集進行漏洞掃描 (Trivy/Checkov)，並提出改進建議。評估是否導入 **Kyverno (Policy-as-Code)** 來實現自動化合規。
3.  **成本優化**：利用 CloudWatch/Azure Cost Management 進行資源盤點，尋找閒置資源或評估 Spot 實例/Fargate 的切換可行性。
4.  **模組化推動**：將重複性的 Terraform 代碼封裝為標準模組，解決「配置漂移 (Config Drift)」問題。

## 🔴 第三階段：價值與轉型 (Day 61 - 90)
**目標：推動關鍵技術落地，提升團隊交付速度。**
1.  **自動化變革**：針對頻繁出錯的手動環節，撰寫 **Ansible** 或 Python 腳本實現端到端自動化。
2.  **Progressive Delivery**：若團隊仍在使用滾動更新 (Rolling Update)，評估並推動 **Argo Rollouts (Canary)** 落地，以降低變更失敗率 (CFR)。
3.  **AIOps 探索**：嘗試導入如 **K8sGPT** 等 AI 工具輔助異常診斷，將故障恢復時間 (MTTR) 二度壓縮。
4.  **知識分享**：舉辦一場內部技術分享 (Technical Brown-bag Session)，將實戰經驗轉化為團隊資產。

## 🔵 跨團隊與廠商管理專項 (External & Vendor Coordination)
**針對管理職能需求，我將重點落實以下工作：**
1.  **廠商交付標準化**：為外包合作夥伴 (Outsourcing Partners) 建立統一的 IaC 與 CI/CD 交付規範，確保第三方開發的代碼符合公司資安標準。
2.  **事故協調機制**：在發生雲端服務異常時，負責與 AWS/GCP 原廠及維護廠商進行技術對接，擔任技術決策中樞。
3.  **合規報告自動化**：定期產出雲端合規與資源使用報告，支援 IT Security 與審計單位的監管需求。

---
> 「我進入這間公司的目標不只是維護，而是要透過 **『工程化手段』** 讓運維變得更可靠、更透明。」
